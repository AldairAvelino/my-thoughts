Documentação da API

Esta é a documentação da sua API, que contém informações sobre os endpoints disponíveis e exemplos de requisições.

Endpoints Disponíveis

  • GET /pensamentos: Retorna todos os pensamentos cadastrados.

    Exemplo de requisição: GET /pensamentos

    Exemplo de resposta:

            
[
    {
      "id": "1",
      "texto": "Conteúdo do pensamento 1",
      "autor": "Autor 1"
    },
    {
      "id": "2",
      "texto": "Conteúdo do pensamento 2",
      "autor": "Autor 2"
    }
]
  • POST /pensamentos: Cria um novo pensamento.

    Exemplo de requisição:

            
POST /pensamentos
Content-Type: application/json

{
  "texto": "Conteúdo do pensamento",
  "autor": "Autor do pensamento"
}

    Exemplo de resposta:

            
{
  "id": "123",
  "texto": "Conteúdo do pensamento",
  "autor": "Autor do pensamento"
}
  • GET /pensamentos/:id: Retorna um único pensamento pelo ID.

    Exemplo de requisição: GET /pensamentos/123

    Exemplo de resposta:

            
{
  "id": "123",
  "texto": "Conteúdo do pensamento",
  "autor": "Autor do pensamento"
}
  • PUT /pensamentos/:id: Atualiza um pensamento existente pelo ID.

    Exemplo de requisição:

            
PUT /pensamentos/123
Content-Type: application/json

{
  "texto": "Novo conteúdo do pensamento",
  "autor": "Novo autor do pensamento"
}

    Exemplo de resposta:

            
{
  "id": "123",
  "texto": "Novo conteúdo do pensamento",
  "autor": "Novo autor do pensamento"
}
  • DELETE /pensamentos/:id: Remove um pensamento pelo ID.

    Exemplo de requisição: DELETE /pensamentos/123

    Exemplo de resposta:

            
{
  "message": "Pensamento removido com sucesso"
}
